**The Problem It Solves:**
Cannabis brands face a unique challenge on social media: they can't directly sell products (Instagram bans this), and they need to navigate strict advertising regulations while still building a compelling brand. Most cannabis marketing feels either too corporate or too stoner-culture focused.

**The Solution:**
Position Smoothie Bar as a "cultural documentarian" of LA cannabis culture rather than just another vape seller. The brand discovers and features emerging LA creatives (artists, musicians, skaters) in their natural environments, showing how cannabis fits into creative lifestyles. This creates engaging content that doesn't feel like traditional advertising while subtly introducing the product's unique dual-strain feature.

**The Unique Angle:**
Smoothie Bar's product has two separate tanks allowing users to experience two different cannabis strains independently or mixed together. The marketing centers on this "blending" concept‚Äîboth literally (mixing strains) and culturally (blending into authentic LA culture).

### **Why This Structure Works**

**Decision Made: HTML for strategy, Markdown for active work**
- HTML documents look polished and professional‚Äîsuitable for sharing with executives or investors
- Markdown files are plain text‚Äîeasy to update quickly as work progresses
- This separation means "finished thinking" vs. "work in progress" are clearly distinguished

**Decision Made: Repeatable template structure**
Every post follows identical format:
1. Metadata (what type of post, which pillar it belongs to, when to publish)
2. Visual strategy (what each image/slide should show)
3. Caption (the actual text that goes with the post)
4. Production notes (special instructions)

This consistency means:
- New team members can quickly understand any post
- Content can be batch-produced efficiently
- Quality stays consistent even with different people creating content

**Decision Made: Living documentation in "Memory Bank"**
The `/memory-bank/` folder contains three critical files that update constantly:
- `activeContext.md` = "What we're working on right now"
- `progress.md` = "What's done, what's next"
- `projectbrief.md` = "Core mission and goals"

These act as the project's working memory‚Äîcapturing decisions and progress so work doesn't get lost or duplicated.

### **Why These Tools Were Chosen**

**HTML for documents**: Can include styling, collapsible sections, and professional formatting while remaining simple files that work anywhere (no special software needed)

**Markdown for tracking**: Universally readable, works with version control systems, easy to edit in any text editor

**Mermaid flowcharts**: Creates visual process diagrams using simple text syntax‚Äîcan be edited without graphic design software

**CSV for comment database**: Standard format that opens in spreadsheets, databases, or can be imported into customer service tools

This is a "simple stack" approach‚Äîprioritizing tools that are accessible, don't require expensive software, and won't break if one team member leaves.
